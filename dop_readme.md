Цель проекта

Это не AI IDE.

Цель проекта — предоставить переносимого автономного помощника, которого можно скопировать на другой компьютер и сразу использовать с минимальной настройкой.

Основные приоритеты:

- переносимость;
- безопасность;
- локальные инструменты;
- автоматизация;
- работа с файлами и системой;
- поиск информации в интернете;
- надёжность.

Проект не стремится конкурировать с AI IDE по количеству функций и не пытается заменить специализированные инструменты для разработки.

Фокус проекта — практическое выполнение повседневных задач: работа с файлами, shell-командами, системой, веб-поиском, документацией, скриптами и автоматизацией в одном переносимом приложении без сложной инфраструктуры и дополнительных сервисов.



Рекомендуется: Ripgrep (rg)

Для более эффективного поиска по файлам, логам, конфигурациям и кодовым базам рекомендуется установить ripgrep (rg).

Агент может использовать "rg" для быстрого поиска файлов, символов, строк и других данных вместо последовательного чтения большого количества файлов.

Скачать

Официальный репозиторий:

https://github.com/BurntSushi/ripgrep

Готовые сборки для Windows доступны в разделе Releases:

https://github.com/BurntSushi/ripgrep/releases

Скачайте архив для Windows ("x86_64-pc-windows-msvc.zip") и распакуйте его.

Вариант 1. Добавить в PATH

Добавьте папку с "rg.exe" в системную переменную "PATH".

Проверить установку:

rg --version

Если команда выполнилась и показала версию — всё настроено правильно.

Вариант 2. Portable (рекомендуется)

Для portable-сборки агента достаточно скопировать файл:

rg.exe

в папку с агентом рядом с исполняемым файлом.

Пример:

PortableAgent/
├── Agent.exe
├── rg.exe
├── .env
├── prompt.txt
└── ...

В этом случае дополнительная установка не требуется.

Зачем это нужно

На средних и больших проектах "rg" позволяет агенту значительно быстрее:

- находить файлы;
- искать функции и классы;
- анализировать кодовые базы;
- искать ошибки в логах;
- находить конфигурации и документацию.

Если "rg" отсутствует, агент продолжит работать в обычном режиме, используя стандартные инструменты файловой системы.
